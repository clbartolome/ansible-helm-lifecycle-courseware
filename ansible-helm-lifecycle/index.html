Welcome to Ansible Helm Lifecycle Lab

Introduction

The aim of this laboratory is to take you through the complete lifecycle of a Helm Chart named base-chart using Ansible Automation Platform. This Helm Chart will help operation teams on daily tasks like:

  • Managing applications deployments

  • Control resources

  • Security and policies

  • Cluster housekeeping

  • Environments and cloud services

The laboratory focus in the lifecycle of the base-chart:

  1. Operations team make a change in base-chart repository (ex: new feature).

  2. Previous change triggers a workflow in Ansible Automation Platform.

  3. The workflow execute a set of job templates to:

    1. Setup testing environment in OpenShift.

    2. Deploy a testing application using the base-chart.

    3. Validate a scenario.

    4. Clean up environment in OpenShift.

    5. Package and upload the base-chart new version to Nexus.

    6. Send a notification with the new version if everything works as expected.

  4. Development teams will update the version of the chart in their application resources (repositories that ArgoCD is using to deploy applications).

  5. ArgoCD will automatically perform the deployment into OpenShift using the new chart.

The lab is divided into two blocks of sections. The first one, from section 1 to 5 is a set of guided exercises that will allow you to familiarize yourself with the environment. The last one is a section called Challenges, a set of exercises with no guide that will allow you to face more real scenarios in your own way.

If you have any question or concern please let us know, we’re more than happy to help!!
